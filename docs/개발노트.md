## 11. 구글시트 연동 설계 (2026-04-19 확정)

### 스키마 (16열, 시트 탭 이름: `상품마스터`)

| # | 열 | 역할 | 타입 | 예시 | 메모 |
|---|----|-----|-----|-----|-----|
| 1 | 브랜드 | 사람 | str | 아디다스 | 필수 |
| 2 | 포이즌 | 사람 | str(V/빈) | V | 등록 플래그 |
| 3 | 크림 | 사람 | str(V/빈) | V | 등록 플래그 |
| 4 | 출고처 | 사람 | str | 정관 | |
| 5 | NAME | 사람 | str | FIREBIRD SHO BLUE | strip |
| 6 | CODE | 사람 | str/int | JD0817 | 고유키 일부 |
| 7 | SIZE | 사람 | str | M | 고유키 일부 |
| 8 | 수량 | 사람 | int | 3 | 0=off |
| 9 | 정상가 | 사람 | int | 59000 | |
| 10 | 할인율 | 사람 | float | 0.3 | 0~1 |
| 11 | **공급가(=하한가)** | 봇 | int | 41300 | `정상가×(1-할인율)` 반올림, 리프라이싱 min |
| 12 | **크림현재입찰가** | 봇 | int | 52000 | |
| 13 | **포이즌현재입찰가** | 봇 | int | 55000 | |
| 14 | **갱신일** | 봇 | str | `2026-04-19 22:30` | KST 텍스트 |
| 15 | **크림상태** | 봇 | str | 정상/off/오류:... | |
| 16 | **포이즌상태** | 봇 | str | 정상/off/오류:... | |

### 설계 원칙
- **공급가 = 하한가 통합**: 별도 하한가 열 없음. 공급가가 리프라이싱 min 역할.
- **공급가 자동 계산**: 봇이 매 사이클 `정상가 × (1-할인율)` 계산해 덮어씀. 사람은 할인율로만 조정.
- **사람 영역(1~10) / 봇 영역(11~16) 공간 분리**: 봇은 11~16 외 절대 안 건드림 (코드 차원).
- **고유키**: `(CODE, SIZE)` 조합.
- **1행 = 안내문구, 2행 = 헤더, 3행~ = 데이터**: gspread `get_all_records(head=2)` 로 읽음.
- **공유 시트 (3명 내외)**: batch_update 로 쓰기 최소화, 변경분만 반영. 사람이 자주 건드리는 열은 절대 쓰지 않음.

### 상태 열 값 규칙 + 현재입찰가 열 연동

| 상태 | 의미 | 현재입찰가 열 |
|------|------|--------------|
| `정상` | 현재 입찰 걸려있음 | 가격 표시 |
| `off` | 의도적 미등록 (수량 0 또는 V 플래그 없음) | 빈칸 |
| `보류` | 사람이 수동 보류 (봇 스킵) | 빈칸 |
| `오류:<사유>` | 시도했으나 실패 | 빈칸 |

가격 열만 훑어도 "지금 실제 올라간 것" 이 한눈에. 빈칸 = 상태 열 확인 트리거.

**조건부 서식 (템플릿에 포함, 구글시트로 변환 시 대부분 유지)**
- `오류:` 접두사 → 배경 빨강 + 빨간 굵은 글자
- `off` → 회색 배경 + 회색 이탤릭
- `정상` → 녹색 글자

### 템플릿 파일
`docs/sheet_template.xlsx` — 1행 안내, 2행 헤더, 3행~ 샘플 5행(뉴에라 2 + 아디다스 3).
- 사람 영역(A~J) 헤더 배경 옅은 파랑, 봇 영역(K~P) 옅은 회색.
- 1행 안내 배경 옅은 노랑, 병합.
- 2행 고정 + 필터. 열 너비 조정.

**사용법**: 구글드라이브 → 새로만들기 → 파일업로드 → 업로드된 xlsx 우클릭 → 구글시트로 열기 → 스프레드시트 ID 를 `.env` 의 `GOOGLE_SHEET_ID` 에 등록.
