
(PowerShell 이면 `set` 대신 `$env:KREAM_EMAIL = "..."`)

## 파일별 용도

| 파일 | 안전? | 동작 |
|------|-------|------|
| `01_목록_조회.py` | ✅ 읽기만 | 내 판매입찰 전부 조회 → 가독성 있게 출력 |
| `02_시세_조회.py` | ✅ 읽기만 | product_id + option 로 시세 / 경쟁최저가 |
| `03_등록.py`     | ⚠️ 실제 등록 발생 | 테스트 파라미터 상수에 안전값 입력 필요 |
| `04_가격수정.py` | ⚠️ 실제 수정 발생 | 이미 올린 ask_id 기반 |
| `05_삭제.py`     | ⚠️ 실제 취소 발생 | 이미 올린 ask_id 기반 |

## 권장 테스트 순서

1. `01_목록_조회.py` — 계정 상태 확인 (현재 어떤 입찰이 올라가 있는지)
2. `02_시세_조회.py` — 테스트할 상품의 시세 확인 (경쟁최저가 파악)
3. `03_등록.py` — **체결 안 될 말도 안 되게 높은 가격**으로 등록
   → 반환된 ask_id 기록
4. `04_가격수정.py` — 3에서 받은 ask_id + 여전히 높은 새 가격
5. `05_삭제.py` — 3의 ask_id 취소

## 안전 수칙

- **실제 돈과 직결**되므로 `03/04` 가격은 반드시 **시세의 3~5배 이상**으로 설정
- 각 스크립트 상단 `설정` 영역의 상수만 수정
- 의심스러우면 `03` 직후 `05` 로 바로 삭제
